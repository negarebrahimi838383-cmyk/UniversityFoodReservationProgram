##توضیح کلی پروژه : 
هدف از ساخت این پروژه ایجاد سامانه ای هوشمند جهت رزرو انلاین غذا برای دانشجویان و اساتید دانشگاه است . 
##اهداف پروژه : 
1_دسترسی اسان دانشجویان به منوی سلف
2_صرفه جویی در زمان کلاسی دانشجویان
3_کاهش هزینه نسبت به غذای آزاد

فریم ورک پیشنهادی :Django
پایگاه داده :MySQL
##مستندات : 
[تحلیل نیازمندی ها]
هدف از طراحی سامانه رزرو غذا برای دانشجویان، ایجاد بستری تحت وب جهت تسهیل فرآیند رزرو، مشاهده منوی غذایی و پرداخت هزینه توسط دانشجویان است.
در این سامانه کاربران (دانشجویان، مدیر سلف و ادمین سیستم) هر یک دارای نقش و دسترسی مشخصی هستند.

نیازمندی‌های کارکردی (Functional Requirements):
	1.	دانشجو باید بتواند در سامانه ثبت‌نام و وارد حساب کاربری خود شود.
	2.	سیستم باید امکان مشاهده منوی غذا در روزهای آینده را فراهم کند.
	3.	کاربر بتواند غذای مورد نظر خود را انتخاب و رزرو نماید.
	4.	امکان لغو رزرو تا زمان مشخصی وجود داشته باشد.
	5.	سیستم باید امکان پرداخت آنلاین هزینه غذا را فراهم کند.
	6.	مدیر سلف بتواند منوی غذایی را مدیریت (افزودن، ویرایش، حذف) کند.
	7.	ادمین سیستم بتواند کاربران، تراکنش‌ها و گزارش‌ها را مشاهده و مدیریت نماید.

نیازمندی‌های غیرکارکردی (Non-Functional Requirements):
	1.	امنیت: اطلاعات کاربران و تراکنش‌ها باید رمزگذاری شود.
	2.	قابلیت اطمینان: سیستم باید در برابر خطا و قطعی مقاوم باشد.
	3.	کارایی: زمان پاسخ‌گویی سامانه باید کمتر از ۳ ثانیه باشد.
	4.	رابط کاربری: طراحی باید ساده و کاربرپسند باشد.
	5.	قابلیت توسعه: سامانه باید به‌گونه‌ای طراحی شود که در آینده بتوان امکانات جدیدی به آن افزود (مثل رزرو غذا برای مهمان)
[موارد کاربردوسناریوها]
بازیگران سیستم (Actors):
	•	دانشجو: کاربر اصلی که رزرو غذا انجام می‌دهد.
	•	مدیر سلف: مسئول ثبت و مدیریت منوی غذایی.
	•	مدیر سامانه (ادمین): مسئول مدیریت کاربران، گزارش‌ها و تنظیمات سیستم
  سناریوی نمونه – رزرو غذا توسط دانشجو:
	1.	دانشجو وارد حساب کاربری خود می‌شود.
	2.	از بخش منوی غذایی، وعده مورد نظر را انتخاب می‌کند.
	3.	سیستم قیمت غذا را نمایش می‌دهد.
	4.	دانشجو بر روی گزینه «رزرو و پرداخت» کلیک می‌کند.
	5.	پس از پرداخت موفق، پیام تأیید رزرو نمایش داده می‌شود.
	6.	اطلاعات رزرو در پایگاه داده ذخیره می‌شود و در صفحه کاربر نمایش داده می‌شود.
  [طراحی سیستم]
  معماری کلی:
سیستم از سه لایه اصلی تشکیل شده است:
	1.	لایه ارائه (Presentation Layer):
رابط کاربری وب برای دانشجویان و مدیران، طراحی‌شده با HTML، CSS و Bootstrap (یا Vue.js/React).
	2.	لایه منطق کسب‌وکار (Business Logic Layer):
پیاده‌سازی با فریم‌ورک انتخابی (مثلاً Django یا Laravel) برای مدیریت عملیات رزرو، پرداخت و کاربران.
	3.	لایه داده (Data Layer):
شامل پایگاه داده رابطه‌ای (MySQL یا SQLite) برای ذخیره اطلاعات کاربران، وعده‌های غذایی، رزروها و پرداخت‌ها.

نمودار کلی پایگاه داده (به‌صورت توضیحی):
	•	جدول Users: شامل اطلاعات دانشجویان (کد دانشجویی، نام، رمز عبور، نوع کاربر)
	•	جدول Meals: اطلاعات مربوط به وعده‌های غذایی (نام، تاریخ، قیمت، ظرفیت)
	•	جدول Reservations: ثبت رزروهای انجام‌شده (کد کاربر، کد غذا، تاریخ رزرو، وضعیت)
	•	جدول Payments: اطلاعات پرداخت‌ها (کد رزرو، مبلغ، وضعیت پرداخت)

طراحی امنیت:
	•	رمزنگاری رمز عبور با الگوریتم مناسب (مثلاً SHA256).
	•	کنترل سطح دسترسی بر اساس نقش کاربر (Student, Admin, Manager).
	•	استفاده از نشست (Session) امن برای ورود کاربران.
